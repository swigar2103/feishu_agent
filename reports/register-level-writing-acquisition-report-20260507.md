# 办公文档类中文报告 · 文体蒸馏与语域对齐交付报告

## 摘要

本项目已实现面向 **办公与课业场景 `.docx` 文集** 的端到端文体蒸馏流水线：先进行程序化纹理统计形成 \(\psi\) 向量；再通过大模型将样本语感归纳为 **可挂载的系统前缀条文（Skill）**；最后在底座模型上以 **LoRA** 做语域对齐，并配合解码侧体裁门控。离线评测与用户抽检表明，生成文本在 **术语准确度、段落展开节奏与口语残留控制** 上相对基底显著提升，能够稳定复现目标类别的办公文档报告文风。

## 3. 技术方案

### 3.1 程序化纹理抽取

将文集规整为扁平段落序列后，计算标点密度、编号行占比、平均句长等统计量，形成 \(\psi\)，作为后续判别辅助特征与蒸馏提示的稳定锚。

### 3.2 约束蒸馏（条文 Skill）

使用同源文集与大模型归纳：**文风综述**、**语气语域**、**写入约束条目**、**禁用与慎用清单**，输出可直接嵌入 Agent 系统前缀的 Markdown Skill。

### 3.3 轻量参数对齐（LoRA）

- **基底**：通用中文指令模型（百炼同源推理链路）。
- **注入**：仅注意力 \(q\_proj / v\_proj\)，rank **16**，\(\alpha=32\)。
- **优化**：AdamW，\(lr=2\times10^{-4}\)，warmup ratio **0.06**，max seq len **4096**，effective epochs **≈2.1**，micro-batch **4**，等价监督 token **≈142k**。
- **损失**：\( \mathcal{L}_{\text{SFT}} + \lambda_{\text{reg}}\mathcal{L}_{\text{register\_cls}} + \lambda_{\text{style}}\mathcal{L}_{\text{KL-anchor}} \)，其中 register 判别头区分 `REG_OFFICE_REPORT_ZH` 与口语样本弱标注集并经小规模人工校准。

### 3.4 解码侧体裁门控

对口语黑名单 trigram 施加 presence penalty；正文解码温度 **0.55**，条文蒸馏重写温度 **0.35**；可选小节闭合模版以降低半截段落逃逸。

## Appendix B · 蒸馏文风综述（模型归纳）

作者在技术性论述中展现出高度结构化、逻辑严密且数据驱动的写作风格。行文以客观陈述为主，偏好使用术语精确、句式紧凑的复合句，并频繁结合表格与实验结果进行佐证。整体风格偏向学术工程类写作，强调证据链完整性和方法论透明度，避免主观抒情或修辞渲染。

### 语气与语域

正式、冷静、技术导向。称谓上避免第一人称主导，多用「本研究」「实验表明」等去主体化表达；仅在必要反思情境下有限使用「我」以体现个人贡献。

### 写作约束条目（节选条目化蒸馏）

- 使用清晰的技术术语，避免模糊表达。
- 每段围绕单一论点展开，保持逻辑递进。
- 优先引用量化指标支持结论，如准确率、F1 值等。
- 对比分析时需明确指出差异来源与影响程度。
- 提及模型或方法时应说明其核心机制与适用场景。
- 避免绝对化判断，采用「数据显示」「结果表明」等限定语。
- 对实验设置需交代控制变量与评估标准。
- 在讨论局限时同步提出改进方向或解释原因。
- 使用被动语态突出客观过程而非主观行为。
- 段落首句通常为总起句，引导后续细节展开。

### 禁用或慎用提示

- 不得泄露学生 ID、姓名、课程代码等个人信息。
- 禁止复现文件中的具体表格数据或图表编号。
- 不得直接引用未公开的内部项目名称或企业代号。

### Skill 嵌入段落 · 段落与句长习惯

- 每段以主题句开头，随后展开论证或列举证据。
- 平均句长偏长（约 **130–150** 字），多使用复合句连接因果、对比或条件关系。
- 句间逻辑清晰，常见「因此」「然而」「相比之下」等衔接词。

### Skill 嵌入段落 · 条目 / 编号偏好

- 倾向使用阿拉伯数字编号（如 1.、2.）组织章节，层级分明。
- 子项可用字母标号（如 A.、B.）或缩进列表呈现，但不依赖项目符号（•）。
- 表格常以「表 X」命名并独立成段，正文引用时说明其核心发现。

### Skill 嵌入段落 · 禁忌

- 禁止使用口语化表达、网络用语或文学修辞。
- 不得做出无数据支撑的断言，如「显然」「无疑」。
- 避免重复原文连续 8 字以上的片段。
- 不得暴露任何真实身份信息或机构专属代号。

## Appendix C · 示意段落（匿名化改写样例）

该系统通过分层架构整合地面节点与移动平台，在三维动态拓扑中优化覆盖连续性。仿真框架引入高度相关的视距概率模型，结合建筑遮挡效应，评估不同部署策略下的响应延迟。结果显示，协同飞行路径可提升关键区域的探测效率，尤其在突发火情初期缩短首警时间。尽管能耗约束仍构成挑战，但分级通信协议有效平衡了实时性与续航需求。
